# Эксплуатация

## Сайт и печать

- Запуск: `python scripts/run_site.py` (логика — `problemgen/web/worksheet_site.py`).
- Быстрый вариант — `generate_random_worksheet(...)`; ручная сборка —
  `generate_combined_worksheet_by_modules(...)`; задач на листе 1–20 (`task-count`).
- Печать: `print-with-answers` — A4 landscape, отрезаемая колонка ответов, QR.

## Template Studio

- Локальная административная страница: `http://127.0.0.1:8090/admin/template-studio`.
- Доступна только loopback-клиенту и требует CSRF для изменяющих JSON-запросов.
- Drafts и отчёты лежат в `data/template_studio/`, а active overlay читает
  `problemgen/web/worksheet_site.py`; production JSON модулей при этом не
  переписывается.
- Краткий маршрут: вставить текст → проверить draft → preview → validate →
  только затем `Add to module`. Подробнее: `Docs/TEMPLATE_STUDIO.md`.

## База знаний (эта БЗ)

- Сервер: `node scripts/kb_server.mjs` → веб-UI http://127.0.0.1:8481
  (loopback-only, noAuthentication; Trilium-checkout ищется автоматически,
  разовая подготовка описана в `Docs/KNOWLEDGE_BASE.md`).
- CLI: `node scripts/kb.mjs help`; зеркало для git — каталог `knowledge/`.
- После `git pull`: `node scripts/kb.mjs sync-in "15min" knowledge`.
- После правок БЗ: `node scripts/kb.mjs sync-out "15min" knowledge` → commit → push.

## Окружение разработчиков

- macOS/Linux: `python`; Windows: `py -3` (в WORK_LOG команды встречаются в обеих
  формах — это одна и та же проверка).
- Возможны git-предупреждения LF→CRLF на Windows — известны, не блокер.
