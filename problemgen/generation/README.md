# problemgen/generation

Здесь должен жить конвейер сборки конкретного экземпляра задачи.

## Подслои

- `selection/`
- `numbers/`
- `binding/`
- `assembly/`

Каждый подслой должен решать свою отдельную часть задачи и не подменять собой соседние слои.

Тематический генератор `digits_templates.py` обслуживает единый модуль цифр,
записи чисел и криптарифмов. Он читает каталог из `data/`, использует только
точную целочисленную арифметику и не выбирает неактивные source-accounting
записи.

Тематический генератор `ratio_templates.py` строит связанные величины модуля
отношений, долей, пропорций и процентов от целого решения и независимо проверяет
точность делений.

Тематический генератор `factor_product_templates.py` обслуживает модуль
множителей, произведений и факториалов: точные пары делителей, факториалы и
подсчёт конечных нулей через показатели степеней 2 и 5.

`combinatorics_templates.py` обслуживает модуль 11: точные размещения,
циклические распределения и упаковку наборов из различных типов ресурсов.

`pigeonhole_templates.py` моделирует худший выбор и точные минимальные гарантии
для трёх категорий и выборок шаров.

`parity_templates.py` классифицирует числа по чётности соседних цифр и точно
суммирует подходящие элементы конечного списка.

`process_templates.py` обслуживает ограниченные Collatz-процессы, циклические
наложения и последовательные суммы модуля 14.

`calendar_templates.py` использует реальный григорианский календарь для дат,
смещений, повторных расписаний и палиндромных записей дат модуля 15.

`clock_templates.py` моделирует цифровые и аналоговые часы строго целыми
секундами, минутами и часами для модуля 16.

`time_zone_templates.py` ведёт маршруты модуля 17 по абсолютной шкале целых минут.

`motion_templates.py` строит связанные скорости, расстояния и durations модуля 18.
