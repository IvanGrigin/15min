# Документация по проекту

## Что это за проект

Проект генерирует математические задачи и сохраняет результаты в `JSON`.

Сейчас в проекте есть два входных файла:

- `01.py` — простой автономный генератор ранней версии.
- `02.py` — основной launcher новой модульной архитектуры.


## Главная идея новой архитектуры

Теперь проект разделен на независимые слои:

1. `problemgen/russian/`
   - единый слой русского языка;
   - словоформы;
   - согласование по числам;
   - базовая валидация текста задачи.

2. `problemgen/core/`
   - общие модели;
   - уровни сложности;
   - общий каталог сюжетных тем.

3. `problemgen/domains/`
   - отдельные блоки математики;
   - внутри каждого блока свои шаблоны и генераторы.

4. `problemgen/web/`
   - локальный HTTP-сервер;
   - HTML-рендеринг;
   - интеграция с фронтендом.

5. `frontend/`
   - отдельные статические файлы интерфейса;
   - CSS;
   - JavaScript;
   - LaTeX-like типографика через `STIX Two Text` и стек `Latin Modern / CMU Serif`.


## Точка входа

### `02.py`

Сейчас это тонкий launcher.

Он больше не содержит доменную логику.

Его задача:

- импортировать `problemgen.cli.main`;
- запускать `CLI` или локальный сервер.


## Пакет `problemgen/`

### `problemgen/cli.py`

Главный интерфейс запуска.

Что делает:

- разбирает аргументы командной строки;
- выбирает домен задач;
- запускает генерацию;
- печатает задачи в консоль;
- запускает локальный сервер по флагу `--serve`.

Основные флаги:

- `--serve`
- `--host`
- `--port`
- `--domain`
- `--count`
- `--template-name`
- `--difficulty-level`
- `--story-theme`
- `--seed-mode`
- `--seed`
- `--mode`
- `--output`


### `problemgen/app.py`

Оркестратор всего приложения.

Что делает:

- собирает каталог доменов;
- вызывает нужный домен;
- нормализует формат задач;
- прогоняет общую проверку русского языка;
- пересохраняет итоговый `JSON` уже после общей валидации.

## Русский язык

### `problemgen/russian/lexicon.py`

Содержит `NounForms` — центральную структуру для словоформ.


### `problemgen/russian/agreement.py`

Содержит:

- `pluralize_ru(...)`
- `count_with_word_ru(...)`
- `join_with_comma_and(...)`
- `normalize_sentence(...)`

Именно сюда должны вноситься исправления, если ошибка касается согласования слов по числу.


### `problemgen/russian/validator.py`

Содержит:

- `LanguageIssue`
- `validate_problem_record(...)`
- `attach_language_report(...)`

Что проверяет:

- пустой текст;
- двойные пробелы;
- начало не с заглавной буквы;
- отсутствие финального знака препинания;
- отсутствие числового ответа;
- потерю единицы измерения в задачах на отрезки.


## Домены задач

### `problemgen/domains/segments/`

Блок задач на отрезки.

#### `domain.py`

Новый адаптер домена.

Что делает:

- подключает legacy-движок;
- описывает шаблоны для нового UI;
- отдает темы и результат в общем формате.

#### `legacy_engine.py`

Это копия прежнего большого `02.py`.

Она сохранена как совместимый движок, чтобы не потерять уже работающие:

- шаблоны задач на прямой и плоскости;
- сюжетные сцены;
- размерности `см / м / км`;
- генерацию задач на отрезки;
- старую внутреннюю логику тем и сложности.

Ключевые legacy-сущности:

- `DIFFICULTY_PROFILES`
- `STORY_THEME_LABELS`
- `TemplateRegistry`
- `SegmentProblemGenerator`
- `generate_problem_set(...)`


### `problemgen/domains/counting/`

Блок задач на счет.

#### `templates.py`

Содержит шаблоны:

- `count_total_groups`
- `count_missing_group`

Они генерируют варианты для `easy / medium / hard`.

#### `domain.py`

Управляет:

- выбором seed;
- выбором темы;
- циклом генерации;
- сохранением в `JSON`.


### `problemgen/domains/combinatorics/`

Блок задач на комбинаторику.

#### `templates.py`

Содержит шаблоны:

- `combo_outfit_pairs`
- `combo_route_pairs`

Оба шаблона используют правило произведения и генерируют задачи разной числовой сложности.

#### `domain.py`

Доменный генератор для комбинаторики.


## Веб-слой

### `problemgen/web/server.py`

Что делает:

- поднимает локальный сервер;
- подбирает свободный порт, если указанный уже занят;
- рендерит HTML;
- раздает `frontend/styles.css` и `frontend/app.js`;
- выдает `JSON` на скачивание;
- показывает задачи, ответы и замечания языковой проверки.

Ключевые сущности:

- `ProblemWebHandler`
- `create_http_server(...)`
- `run_server(...)`
- `render_problem_cards(...)`
- `render_page(...)`


## Фронтенд

### `frontend/styles.css`

Отвечает за внешний вид:

- светлая бумажная палитра;
- LaTeX-like serif typography;
- адаптивная сетка;
- карточки задач;
- подсветка ошибок проверки русского языка.


### `frontend/app.js`

Отвечает за интерактивность формы:

- подменяет список шаблонов при смене домена;
- подменяет список тем при смене домена;
- использует конфигурацию, встроенную сервером в HTML.


## Как сейчас устроена сложность

Есть два уровня сложности:

1. Общий пользовательский уровень:
   - `easy`
   - `medium`
   - `hard`

2. Доменный числовой уровень:
   - каждый домен сам задает диапазоны чисел для своих шаблонов;
   - при росте чисел одна и та же задача становится сложнее вычислительно.

Примеры:

- в `counting` и `combinatorics` диапазоны лежат в `templates.py` соответствующего домена;
- в `segments` числовая логика по-прежнему лежит в `legacy_engine.py`.


## Как сейчас устроен JSON

Итоговый `JSON` содержит:

- условие;
- ответ;
- код задачи;
- категорию;
- домен;
- шаблон;
- сюжет;
- сложность;
- метаданные;
- результат проверки русского языка.


## Типичные запуски

### Консоль, отрезки

```bash
python3 02.py --domain segments --count 5 --difficulty-level medium --story-theme ants
```

### Консоль, счет

```bash
python3 02.py --domain counting --count 5 --difficulty-level easy --story-theme space
```

### Консоль, комбинаторика

```bash
python3 02.py --domain combinatorics --count 5 --difficulty-level hard --story-theme cowboy
```

### Локальный сайт

```bash
python3 02.py --serve
```

Если порт занят, сервер автоматически попробует следующие порты.


## Что открывать в зависимости от задачи

- если нужно менять общий запуск: `problemgen/cli.py`
- если нужно менять маршрутизацию доменов: `problemgen/app.py`
- если нужно чинить русский язык во всех новых задачах сразу: `problemgen/russian/`
- если нужно добавлять новый блок математики: `problemgen/domains/`
- если нужно менять UI: `problemgen/web/server.py`, `frontend/styles.css`, `frontend/app.js`
- если нужно чинить старые задачи на отрезки: `problemgen/domains/segments/legacy_engine.py`
