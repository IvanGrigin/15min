# data/templates

Здесь лежат формализованные шаблоны задач.

## Что должен содержать шаблон

- `template_id`;
- `source_ref`;
- математическую структуру;
- `text_pattern` со слотами;
- роли сущностей;
- числовые ограничения;
- признаки решения;
- feature flags;
- жанр, категорию, подкатегорию и сложность.

## Как это связано с кодом

`problemgen/catalog/` должен читать этот слой.

`problemgen/generation/` должен брать отсюда контракт шаблона и наполнять его числами, сущностями и языковыми формами.

## Дополнительные подкаталоги

- `worksheets/` — JSON-шаблоны визуального оформления листов с задачами для PDF/PNG-рендера.
