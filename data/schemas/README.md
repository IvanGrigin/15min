# data/schemas

Здесь лежат схемы и примерные контракты данных.

## Что должно быть в этой папке

- примеры структуры шаблона задачи;
- примеры структуры экземпляра задачи;
- примеры структуры сущности;
- примеры структуры языковой записи, например глагола;
- при необходимости новые схемы для bundle, trace и validation report.

## Зачем это нужно

Эти файлы нужны, чтобы будущий агент видел не только код, но и ожидаемый формат данных до реализации конкретных загрузчиков и валидаторов.
