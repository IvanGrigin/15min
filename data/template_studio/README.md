# Template Studio

Здесь хранится только административный слой создания новых параметрических
шаблонов. Он изолирован от `data/templates/problem_sets/*`: анализ никогда не
изменяет production-каталоги автоматически.

Структура:

- `drafts/` — один JSON-файл на незавершённый, validated, archived или rejected
  черновик;
- `history/` — журнал важных действий и атомарные снимки active-overlay перед
  его изменением;
- `validation_reports/` — последний подробный отчёт проверки каждого черновика;
- `active_templates.json` — единственный overlay-каталог валидированных и явно
  активированных шаблонов. Его читает сайт; archived-записи в него не попадают.

Файлы создаются только через `problemgen.template_studio.storage.TemplateStudioStore`.
Он записывает JSON во временный файл и заменяет целевой файл через `os.replace`,
поэтому неполная запись не становится рабочим каталогом.

Черновики не являются исходниками задач и не должны вручную переноситься в
каталоги модулей. Для публикации используйте страницу `/admin/template-studio`.
