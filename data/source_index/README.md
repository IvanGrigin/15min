# data/source_index

Здесь лежит структурированная разметка по `Docs/all_tasks_all_files.md`.

Для повседневной навигации по корпусу начинать с `task_tree/README.md`.
Там 100 тематических листьев и `manifest.json`; каждый лист намеренно
небольшой и объединяет таксономию, план формализации и записи корпуса.

## Что должно быть в этой папке

- индекс отдельных исходных задач;
- группировка близких задач по математическому типу;
- пометки о сложности и пригодности к параметризации;
- ссылки на целевые шаблоны после их создания.

## Главное правило

Это производный слой.

Он может дополняться, но не заменяет собой исходный read-only файл и не дает права его переписывать.
