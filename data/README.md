# data

Эта папка хранит проектные данные, а не исполняемый код.

## Что должно лежать здесь

- схемы и примеры JSON-структур;
- таксономии категорий, жанров и feature flags;
- индекс исходных задач;
- формализованные шаблоны задач;
- словари сущностей;
- словари глаголов, предлогов и правил согласования.

## Что не должно лежать здесь

- Python-логика генерации;
- web-слой;
- одноразовые скрипты;
- результаты генерации.

## Как это связано с кодом

Код из `problemgen/` должен читать этот слой через явные загрузчики и не должен дублировать здесь лежащие словари в виде hardcoded-констант без веской причины.

## Важное правило

Комментарии в `README.md` и поясняющие `_comment` в `*.example.json` считаются частью архитектурного контракта. Не удалять их без замены более точным описанием.
