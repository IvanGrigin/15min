# Модуль 11: комбинаторика и подсчёт вариантов

ID `combinatorics_and_counting_variants`, английское название “Combinatorics
and Counting Variants”. Источники — два read-only файла `Docs/11_...md`: пять
задач без имён и четыре именованных. Из исходных 10 записей удалён один дубль,
поэтому manifest учитывает 9 уникальных номеров.

Runtime-каталог `data/templates/problem_sets/combinatorics_and_counting_variants/templates.json`
содержит четыре семейства: нечётные числа в открытом интервале; упорядоченные
непустые выборки различимых объектов; циклическую выдачу с чередованием;
максимальную упаковку наборов из разных типов ресурсов. Восемь источников
активны. Задача 1205 исключена: она требует описания логической процедуры, а не
единственного целого ответа.

`problemgen/generation/combinatorics_templates.py` использует точные формулы и
проверяет малые случаи перебором. Для именованных семейств персонаж выбирается
из approved-каталога; имя остаётся целиком в именительном падеже, а русские
числительные используют общий `pluralize_ru`. Fixed seed воспроизводит выбор и
параметры. Сайт получает структурированный integer-ответ отдельно от условия.

Проверка: `py -3 scripts/validate_combinatorics_templates.py`,
`py -3 -m unittest tests.test_combinatorics_templates tests.test_worksheet_site`.
Новое семейство требует semantic ID, one-to-one записи manifest, ограниченной
стратегии, независимой проверки и не менее 20 seed-тестов.
