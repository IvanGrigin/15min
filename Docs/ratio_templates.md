# Шаблоны отношений, долей, пропорций и процентов

## Контракт модуля

- ID: `ratios_fractions_proportions_and_percentages`.
- Русское название: «Отношения, доли, пропорции и проценты».
- Источники: два read-only файла `docs/10_..._deduplicated.md`.
- В исходном модуле было 35 записей; после удаления 9 дублей осталось 26 задач.

## Файлы и поток данных

`data/templates/problem_sets/ratios_fractions_proportions_and_percentages/templates.json`
хранит 12 активных семейств. `source_accounting.json` отдельно учитывает все
26 источников. Код `problemgen/generation/ratio_templates.py` загружает каталог,
строит связанный набор параметров, повторно решает задачу точной целочисленной
арифметикой и возвращает структурированный результат сайту.

Активные семейства: максимум различных положительных частей; момент половины
при удвоении; сумма начальной части треугольного ряда; три величины по двум
разностям; отношение долей в процентах; распределение с двумя передачами;
повторное деление пополам; повтор дня недели для фиксированной даты;
последовательные доли остатка; целочисленное уравнение стаи; сохранение общей
суммы; минимальный состав по процентному порогу.

18 источников активны. 8 исключены: четыре требуют составного или
конструктивного ответа (635, 641, 847, 1312), два дают текстовую раскладку или
имя победителя (752, 779), два неоднозначны без дополнительных правил выбора
или расположения (503, 1092). Точная причина хранится у каждой записи manifest.

## Точность и язык

Значения строятся назад от целого решения. Деления проверяются через остаток,
проценты считаются только целыми числами, а генерация ограничена 80 попытками с
диагностикой ID и seed. Для именованных задач персонажи выбираются только из
`docs/approved_dimensions_150_characters.md`: одна вселенная, разные роли.
Формулировки оставляют полные, в том числе составные и нарицательные, имена в
именительном падеже и используют безопасное множественное согласование. Формы
числительных выбираются общим `pluralize_ru`.

## Интеграция и проверка

Модуль зарегистрирован в `catalog.json` и общем worksheet-site. Слот выбирает
модуль, а генератор случайно выбирает один из 12 активных шаблонов; fixed seed
воспроизводит и выбор, и параметры. Ученический лист получает только условие,
ключ ответов — точное целое значение.

```powershell
py -3 -m unittest tests.test_ratio_templates
py -3 -m unittest tests.test_worksheet_site
py -3 scripts/validate_ratio_templates.py
py -3 -m unittest discover -s tests
py -3 scripts/run_site.py
```

Чтобы безопасно добавить семейство, нужно внести уникальный semantic ID в
`templates.json`, связать все его источники ровно один раз в manifest,
зарегистрировать ограниченную стратегию и независимый решатель, добавить не
менее 20 seed-проверок, затем обновить эту страницу, индексы и журнал.
