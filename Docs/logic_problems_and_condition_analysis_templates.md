# Модуль 30. Логические задачи и анализ условий

## Источники и учёт

Используются только read-only файлы `30_logicheskie_zadachi_i_analiz_usloviy_*_deduplicated.md`:
5 задач без имён и 32 с индивидуализированными действующими лицами. Из 48
исходных записей после удаления 11 дублей осталось 37 уникальных номеров.
`source_accounting.json` покрывает каждый из них ровно один раз: 9 активных
номеров и 28 исключённых (текст, доказательство, yes/no, несколько ответов,
неоднозначная конфигурация или недостающий чертёж).

## Активные семейства

1. Одна ошибка в трёх связанных произведениях — ответом служит единственный
   правильный результат ошибившегося ученика.
2. Круг рыцарей и лжецов — конфигурации исчерпывающе проверяются по всем
   булевым состояниям, принимаются только параметры с единственным числом
   лжецов.
3. Высказывание «все остальные лжецы» — независимая логическая проверка
   оставляет ровно одного правдивца.

Все runtime-ответы — `int`; генерация ограничена 50 попытками и возвращает ID
шаблона и seed при ошибке. В JSON нет Python-выражений, исключённые записи не
участвуют в выборе.

## Язык и персонажи

Семейство с произведениями выбирает трёх разных персонажей одной вселенной из
`docs/approved_dimensions_150_characters.md`. Учитель выдаёт им карточки с
примерами; имена используются только в именительном падеже в записи
«имя: выражение», поэтому нет небезопасного склонения составных и нарицательных
имён и ложной роли «ученик».

## Интеграция и проверки

Каталог: `data/templates/problem_sets/logic_problems_and_condition_analysis/`.
Генератор: `problemgen/generation/logic_templates.py`. Сайт регистрирует модуль
в общем selector и передаёт ответы только в ключ.

```powershell
py -3 -m unittest tests.test_logic_templates tests.test_worksheet_site
py -3 scripts/validate_logic_templates.py
```

Новый шаблон добавляют только вместе с записью в manifest, независимым solver,
seed-проверками и регистрацией в `worksheet_site.py`.
