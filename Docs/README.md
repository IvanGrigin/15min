# Docs

Папка `Docs/` хранит:

- служебную документацию проекта;
- архитектурные blueprint-документы;
- журнал и навигацию;
- read-only корпус исходных задач `all_tasks_all_files.md`.

## Основные файлы

- `ARCHITECTURE_BLUEPRINT.md` — целевая архитектура и границы слоев.
- `REPOSITORY_STRUCTURE.md` — что должно лежать в каждой папке.
- `DATA_FLOW.md` — путь данных от исходного корпуса до итогового JSON.
- `FILE_INDEX.md` — индекс ключевых файлов проекта.
- `VECTOR_TREE.md` — короткая смысловая навигация.
- `WORK_LOG.md` — журнал изменений.
- `all_tasks_all_files.md` — основной read-only источник типов и примеров задач.
- `source_documents/` — неизменённые импортированные источники, ожидающие отдельной индексации.

## Как этим пользоваться

Если нужно быстро войти в проект:

1. открыть `AGENTS.md`;
2. открыть `Docs/ARCHITECTURE_BLUEPRINT.md`;
3. открыть `Docs/DATA_FLOW.md`;
4. открыть `Docs/VECTOR_TREE.md`;
5. открыть нужный раздел в `Docs/FILE_INDEX.md`;
6. посмотреть последние записи в `Docs/WORK_LOG.md`.

## Важное правило

`Docs/all_tasks_all_files.md` нельзя редактировать.

Если нужно добавить структурированную разметку по исходным задачам, ее нужно класть в `data/source_index/` или `data/templates/`, а не в этот файл.
