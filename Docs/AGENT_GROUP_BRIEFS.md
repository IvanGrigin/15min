# Брифы и промпты по группам

Готовые промпты для каждой свободной группы + предметный комментарий: какая
математика, какие функции движка понадобятся, как сверять ответы, где подводные
камни. База — [TEMPLATE_AUTHORING_GUIDE.md](TEMPLATE_AUTHORING_GUIDE.md),
[AGENT_TASKS.md](AGENT_TASKS.md), доска — [AGENT_STATUS.md](AGENT_STATUS.md).

## Два обязательных правила (уроки Группы D)

1. **Отдельная ветка без новой папки.** Не создавать `git worktree`, клон или
   временную папку. Каждый агент в существующем корне репозитория создаёт ветку:
   ```bash
   git fetch origin
   git switch -c authoring/<группа> origin/main
   ```
   После тестов агент делает rebase, push ветки и merge в `main` через PR. В одном
   общем каталоге агенты выполняются последовательно, а не параллельно.
2. **Ответ проверять независимо, а не только тестами.** Штатные тесты проверяют лишь
   ТИП ответа. Для каждого шаблона напиши отдельный brute-force расчёт из условия и
   сверь на 20–40 сидах. Формулу и текст держи парными: варианты формулировки
   («сколько» vs «наибольшее») — это РАЗНЫЕ шаблоны с разными формулами.

Уже есть в движке: `abs, min, max, gcd, lcm, isqrt, comb, perm, digit_sum,
count_digit, count_multiples, num_divisors, weekday_after, bigger_label` (+ helper'ы
группы D). Новые функции добавляй в `_FUNCTIONS` с префиксом группы и обязательно
сверяй с эталоном.

## Группа B — Алгебраическое моделирование (302 задачи, средне)

Темы: линейные уравнения «найди x» (B01), возраст (B02), отношения (B03), займы/
взаиморасчёты (B04), головы-ноги (B05), системы цен (B06), совместная работа (B07),
проценты и концентрации (B08), прямая пропорция (B09), передача денег (B10).

Комментарий:
- Многие типы уже реализованы в ручных модулях (`joint_work`, `ages`, `ratios`,
  `heads_and_legs`, `price_system`, `equal_payment`, `movement`) — переиспользуй
  стратегии, не изобретай заново.
- **B01 «найди x»**: генерируй уравнение ИЗ ответа — выбери целое x, построй
  выражение, подставь. Так ответ гарантированно целый; сверка = подставить x обратно.
- B08 проценты: держи числа кратными, чтобы ответ был целым.
- Ответы почти везде `number`. Новые функции движка не нужны.

## Группа E — Последовательности (139 задач, средне)

Темы: арифметические прогрессии и суммы (E01), знакочередующиеся/блочные суммы (E02),
закономерности (E03), Фибоначчи (E04), итерации над числом (E05), рекурсивные суммы (E06).

Комментарий:
- E01: сумма АП = n·(a1+an)/2 — чистая формула.
- E02/E06: для блочных и рекурсивных сумм заведи helper (напр. `e_seq_sum(...)`),
  считающий сумму по правилу; сверяй прямым суммированием.
- E04: добавь `fib_term(n)`; E05: итерации — маленький solver-helper.
- Ответы `number`. Осторожно с границами (сумма не должна взрываться).

## Группа F — Комбинаторика (160 задач, средне)

Темы: перестановки (F01), с повторениями (F02), слова длины k (F03), неизвестный
алфавит (F04), выборки/команды (F05), круговые турниры (F06), на выбывание (F07),
маршруты по клеткам (F08), маршруты с весом (F09), шахматные расстановки (F10),
принцип Дирихле (F11).

Комментарий:
- `comb`, `perm` уже есть. F08 «вправо-вверх» = `comb(m+n, n)`. F06 = `comb(n,2)`.
  F07 на выбывание = n−1 игр.
- F02 перестановки с повторениями: добавь `perm_multiset(...)` (факториал / произв.
  факториалов); нужен `factorial`.
- F11 Дирихле: чистая арифметика (worst case + 1).
- Сверка — прямым перебором на малых значениях. F04/F10 сложнее — может нужен
  специальный счётчик; помечай в `notes`.

## Группа G — Планиметрия и измерения (188 задач, средне)

Темы: площадь/периметр квадрата (G01) и прямоугольника (G02), составные фигуры (G03),
периметр после разреза (G04), целочисленные стороны (G05), масштаб площади (G06),
перевод единиц (G07), подсчёт квадратов на сетке (G08), точки на прямой (G09, G10),
высота слоя жидкости (G11), отношения сторон (G12).

Комментарий:
- Базовые формулы (S=a², P=4a и т.д.) — арифметика.
- **G01 часто спрашивает ДВА случая** («если периметр 20… а если 34») → `answer_type
  = multi`, формула `[case1, case2]`.
- G05 «целочисленные стороны при условии» и G08 «число квадратов на сетке»
  (`n(n+1)(2n+1)/6`) — заведи helper и сверь перебором.
- G07 единицы: следи за коэффициентами перевода, ответ держи целым.

## Группа H — Сетки и объёмная геометрия (158 задач, СЛОЖНО)

Темы: перегородки в решётке (H01) и фигурах с вырезами (H02), клетчатые буквы (H03),
распилы досок (H04), бруски после распила куба (H05), сравнение распилов (H06),
окрашенный куб (H07), расход краски (H08), подписи на гранях кубиков (H09).

Комментарий:
- H04 распилы = (число частей − 1); H05/H06 = отношение объёмов; H08 уже есть аналог
  `paint_cube`.
- **H01/H02 (перегородки в фигуре с вырезом) — самый сложный тип** (это как задача №1
  корпуса). Нужен геометрический solver-helper; вывести формулу и тщательно сверить.
  Если не выводится чисто — помечай `notes`, бери сначала простые листья (H04–H08).
- H07/H09 окрашенный куб: формулы для угловых/рёберных/граневых кубиков.

## Группа I — Время, календари, часы (295 задач, СЛОЖНО, нужен движок)

Темы: день недели через N дней (I01), число дней недели в месяце (I02), n-й день
недели (I03), часовые пояса (I04, I05), разворот в пути по местному времени (I06),
спешащие/отстающие часы (I07), табло и стрелочные часы (I08).

Комментарий:
- Текущий `weekday_after(start, days)` берёт числовой стартовый день. Но **корпус
  стартует от РЕАЛЬНОЙ даты** («2 апреля 2026»). Добавь календарные функции:
  `weekday_of_date(y, m, d)`, `days_in_month(y, m)` (с високосными), счётчик
  вхождений дня недели в месяце. Обязательно сверь с `datetime`/`calendar`.
- Ответы часто `text` (название дня). I07/I08 (часы) — беспоспоставные solver'ы,
  бери их последними.
- Начинай с I01–I03 (даты), они закрываются календарными функциями.

## Группа J — Движение и скорости (122 задачи, средне)

Темы: путь-скорость-время (J01), встречное (J02), вдогонку (J03), три участника (J04),
туда-обратно (J05), средняя скорость (J06), по кругу (J07).

Комментарий:
- Базис d=v·t; частично уже есть `movement`, `opposite_motion` — переиспользуй.
- **J06 средняя скорость = общий путь / общее время** (НЕ среднее скоростей!). Подбирай
  числа так, чтобы деление было точным (ответ целый).
- Ответы `number`. J04/J07 (скрытые расстояния, круг) сложнее — помечай `notes`.

## Группа K — Логика, инварианты, алгоритмы (159 задач, СЛОЖНО)

Темы: рыцари/лжецы (K01), ровно одна ошибка (K02), лифт и достижимость (K03),
чётностные инварианты (K04), разности соседей (K05), состав набора по выборкам (K06),
логическое восстановление (K07), преобразования состояний (K08).

Комментарий:
- **K02 «ровно одна ошибка» — по сути уравнение «найди x»** → арифметика, как B01.
- K03/K08 достижимость часто сводится к `gcd`; K04/K05 инварианты чётности → ответ
  `text` («можно/нельзя») по чёткому правилу (helper, возвращающий строку).
- K01/K07 (словесные логические головоломки) — фиксируй структуру шаблона и логику
  solver'ом; бери их последними. Начинай с K02/K03/K06.

## Как выглядит промпт для агента

Базовый промпт — в [AGENT_TASKS.md](AGENT_TASKS.md). К нему добавляется блок «Особенности
группы <X>» из соответствующего раздела выше и два обязательных правила (ветка без
новой папки +
независимая сверка). Пример готового промпта — в ответе, которым сопровождён этот файл.
