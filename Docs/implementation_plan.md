# План дальнейшей реализации

## 1. Текущая структура проекта

- `Docs/` и `docs/` — документация, архитектурные записи и неизменяемый корпус исходных задач.
- `data/` — индексы исходных задач, JSON-шаблоны, языковые данные и заготовки словарей сущностей.
- `problemgen/` — исполняемая логика доменов, генерации, рендеринга и локального HTTP-слоя.
- `scripts/` — короткие команды ручного запуска.
- `frontend/` — JavaScript и CSS локального интерфейса.
- `outputs/` — созданные JSON, PDF, PNG и HTML-артефакты.
- `tests/` — автоматические проверки.

## 2. Где хранится дерево задач

- Основной read-only корпус: `Docs/all_tasks_all_files.md`; его нельзя редактировать.
- Тематическое дерево корпуса: `data/source_index/all_tasks_tree_by_theme_and_difficulty.md`.
- Интерактивное представление дерева: `outputs/generated/all_tasks_tree_view.html`.

## 3. Где хранятся персонажи и локации

- Текущий рабочий список миров, персонажей и локаций: `problemgen/core/story_worlds.py`.
- Будущий слой данных для нормализованных сущностей: `data/entities/characters/` и `data/entities/locations/`; сейчас там находятся примеры структуры, а не полный рабочий каталог.

## 4. Где находится генерация задач

- Старые арифметические фабрики: `problemgen/domains/arithmetic/templates.py`.
- Новый JSON-шаблонный поток: `data/templates/problem_templates.json`, `problemgen/catalog/problem_templates.py` и `problemgen/generation/template_generator.py`.
- Сборка листа из пяти задач: `problemgen/worksheet/service.py`.
- Команды запуска: `scripts/generate_problem_from_template.py`, `scripts/generate_problem_set.py` и `scripts/generate_worksheet.py`.

## 5. Где находится рендеринг листа

- JSON-разметка листа: `data/templates/worksheets/worksheet_5_tasks.json`.
- PDF/PNG-рендерер: `problemgen/io/worksheet_renderer.py`.
- Локальный HTTP-обработчик существующего интерфейса: `problemgen/web/worksheet_site.py`.

## 6. Файлы для следующего шага

Следующий этап должен затронуть только следующие файлы или их прямые тесты:

- `data/templates/problem_templates.json` — добавить или уточнить один небольшой пул шаблонов выбранного модуля;
- `problemgen/generation/template_generator.py` — только если этому модулю нужна новая числовая стратегия;
- `tests/test_template_generator.py` — тесты нового или уточнённого шаблона;
- `Docs/PROBLEM_TEMPLATES.md` — краткое описание добавленного модуля.

Не следует менять `Docs/all_tasks_all_files.md`, лист, сайт или legacy-генераторы до отдельного решения.

## 7. Безопасный план реализации

1. Выбрать один модуль из дерева задач, например `movement` или `joint_work`.
2. Согласовать математическую модель: переменные, диапазоны, формулу и требование целого ответа.
3. Добавить 2-3 статичных JSON-шаблона только для этого модуля.
4. При необходимости добавить одну общую `number_strategy`; не дублировать шаблонные тексты в Python.
5. Написать тесты на уникальность шаблонов, ограничения чисел, целый ответ и отсутствие незаполненных плейсхолдеров.
6. Сгенерировать несколько примеров вручную и проверить русский текст.
7. Только после проверки одного модуля повторить этот процесс для следующего.

Такой порядок позволяет развивать генератор по частям, не переписывая проект, не создавая массовые наборы шаблонов и не затрагивая существующий интерфейс листа.
