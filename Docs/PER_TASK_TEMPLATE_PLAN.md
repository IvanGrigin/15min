# План: точный шаблон и формула ответа под каждую задачу корпуса

## Цель

Сделать так, чтобы генератор мог воспроизводить **все 2121 задачи** из read-only
корпуса `Docs/all_tasks_all_files.md` и порождать похожие на них варианты. Для
этого под каждую отдельную задачу нужен свой шаблон (`template_text`), своя
стратегия подбора чисел (`number_strategy`) и своя формула ответа (`answer_formula`).

Это отличается от текущего состояния каталога: сейчас 900 bridge-шаблонов — это
заглушки-«мостики» (часто вообще не того математического типа, что тема листа), а
59 ручных шаблонов покрывают лишь несколько тем. Настоящее покрытие «как в
корпусе» — это ручная работа по каждой задаче, и данный план её структурирует.

## Рабочие файлы

- `data/source_index/per_task_template_worklist.json` — **ворклист**: по одному
  слоту на каждую из 2121 задач. Заполненные вручную поля и есть результат работы.
- `scripts/build_per_task_template_worklist.py` — генератор ворклиста. Сводит текст
  задачи из корпуса с темой/сложностью из дерева. Идемпотентен: повторный запуск
  обновляет только «известные» поля (текст, тема, сложность) и **не затирает**
  слоты со `status != "todo"`.

## Структура слота

| Поле | Кто заполняет | Смысл |
|---|---|---|
| `corpus_id` | авто | номер задачи 1..2121 |
| `leaf_id`, `leaf_title` | авто | тема из дерева `task_tree` |
| `difficulty`, `quality` | авто | из разметки листа |
| `source_file`, `source_line` | авто | где лежит оригинал |
| `original_text` | авто | дословный текст задачи из корпуса |
| `answer_engine_hint` | авто (эвристика) | `arithmetic_candidate` или `needs_extension` |
| `proposed_template_id` | авто | предлагаемый id, напр. `task_0002_B01_v1` |
| `template_text` | **вручную** | условие с плейсхолдерами |
| `number_strategy` | **вручную** | имя функции подбора чисел |
| `answer_formula` | **вручную** | формула ответа |
| `answer_type` | **вручную** | `number` / `multi` / `text` |
| `notes`, `status` | **вручную** | заметки и `todo → in_progress → done` |

## Две дорожки

Эвристика в скрипте делит задачи на две группы (счётчики — в шапке ворклиста):

### 1. `arithmetic_candidate` (~1527 задач) — делаются на текущем движке
Ответ выражается через `+ − × / ±` и одно число. Порядок работы:
1. Разобрать оригинал: какие числа переменные, какие фиксированы.
2. Написать `template_text` с плейсхолдерами, сохранив структуру условия.
3. Добавить/переиспользовать `number_strategy` (`@_number_strategy` в
   `problemgen/generation/template_generator.py`), подбирающую числа так, чтобы
   ответ был целым и в разумных границах.
4. Записать `answer_formula`.
5. Прогнать генерацию на многих сидах, убедиться в целом ответе и корректном тексте.

### 2. `needs_extension` (~594 задачи) — движок расширен, группа разблокирована
Это задачи, где ответ раньше нельзя было получить простой арифметикой: подсчёт
вхождений цифры, день недели, делимость/НОД/остатки, комбинаторный подсчёт,
«какое больше и на сколько» (два ответа) и т. п.

**Движок расширен** (`evaluate_formula` в `problemgen/generation/template_generator.py`):

- операторы `//`, `%`, `**` (степень ограничена целым показателем 0..64);
- белый список функций: `abs, min, max, gcd, lcm, isqrt, comb, perm, digit_sum,
  count_digit, count_multiples, num_divisors, weekday_after` — всё вне списка
  (например `open`, `eval`) по-прежнему отклоняется;
- `answer_type`: `"number"` (по умолчанию), `"multi"` (список из нескольких
  ответов), `"text"` (строка, например день недели);
- строки-константы в формуле.

Готовые эталоны каждого класса уже в каталоге: `digit_count_range_001`
(подсчёт цифры), `gcd_pair_001` (НОД), `weekday_after_001` (текстовый ответ),
`two_products_compare_001` (ответ из двух частей).

Порядок такой же, как для арифметики, только `answer_formula` может вызывать
функции из белого списка, а `answer_type` выставляется в `multi`/`text` при
необходимости. Если задаче нужна функция, которой ещё нет, её добавляют в
`_FUNCTIONS` (одна строка) — это и есть точка роста движка.

Остаточно тяжёлые случаи (нестандартная геометрия с построением, хитрые
инварианты) помечать в `notes`; для них может понадобиться отдельный solver.

## Порядок ведения работы

1. При изменении корпуса или дерева — перегенерировать ворклист:
   `python scripts/build_per_task_template_worklist.py`.
2. Брать задачи батчами (например, по одному листу `leaf_id` за раз — так
   переиспользуется одна `number_strategy` на группу похожих задач).
3. Готовый шаблон дублировать в рабочий каталог `data/templates/problem_templates.json`
   (ворклист — источник истины по замыслу, каталог — то, что реально грузит генератор).
4. Обновлять `status` на `done` только после успешной проверки генерации.

## Критерий завершённости

- Все 2121 слота имеют `status = "done"`.
- Для каждого слота генератор порождает и оригинальную структуру, и валидные
  варианты (целый/корректный ответ, без незаполненных плейсхолдеров).
- `python -m unittest discover -s tests` — зелёный, включая проверку, что каждый
  шаблон устойчив на множестве сидов (см. заметку о надёжности в `Docs/WORK_LOG.md`).
