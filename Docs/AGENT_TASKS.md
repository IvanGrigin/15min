# Задачи для агентов: замена 900 заглушек авторскими шаблонами

Главный рычаг проекта — заменить 900 bridge-заглушек (генерируют не тот тип
математики) авторскими шаблонами по темам, чтобы задачи стали такими же, как
оригиналы 15-минуток. Работа разбита так, чтобы **разные агенты вели разные
группы тем без конфликтов**.

Перед работой агент читает [TEMPLATE_AUTHORING_GUIDE.md](TEMPLATE_AUTHORING_GUIDE.md).

## Как разбита работа

Дерево из 100 листьев поделено на 11 тематических групп (A–K). **Один агент = одна
группа.** Внутри группы агент идёт по листьям батчами. Прогресс — в
`data/source_index/per_task_template_worklist.json` (`status = done`).

| Агент | Группа | Тема | Листья | Задач | Готово |
|---|:--:|---|--:|--:|--:|
| A | A | Арифметика и выражения | 8 | 188 | 23 |
| B | B | Алгебраическое моделирование | 10 | 302 | 0 |
| C | C | Цифры и позиционная запись | 12 | 280 | 11 |
| D | D | Теория чисел | 9 | 130 | 0 |
| E | E | Последовательности и закономерности | 6 | 139 | 0 |
| F | F | Комбинаторика | 11 | 160 | 0 |
| G | G | Планиметрия и измерения | 12 | 188 | 0 |
| H | H | Сетки и объёмная геометрия | 9 | 158 | 0 |
| I | I | Время, календари, часы | 8 | 295 | 0 |
| J | J | Движение и скорости | 7 | 122 | 0 |
| K | K | Логика, инварианты, алгоритмы | 8 | 159 | 0 |

Списки листьев каждой группы — в `data/source_index/task_tree/<группа>_*/` и в
`manifest.json`.

## Правила координации (чтобы агенты не мешали друг другу)

1. **Владение по группе.** Агент трогает только листья своей группы: свои записи в
   ворклисте (`leaf_id` начинается с буквы группы) и заглушки своих листьев
   (`source_tree_leaf` той же буквы).
2. **Общий файл — с осторожностью.** `problem_templates.json` и
   `template_generator.py` общие. Каждый агент дописывает свои шаблоны/стратегии и
   удаляет только свои `tree_<своя буква>*` заглушки. Именовать стратегии и
   `template_id` с префиксом темы, чтобы не пересекаться.
3. **Отдельная ветка на агента:** `authoring/<группа>` — напр. `authoring/D`. Мерж
   в `main` по мере готовности батчей. Так правки общих файлов разводятся по PR.
4. **Маленькие батчи, частые коммиты.** Один лист (или под-семейство листа) = один
   коммит. Так конфликты в общих файлах минимальны и легко ревьюятся.
5. **Тесты перед коммитом:** `python -m unittest discover -s tests` — зелёно,
   включая страж хрупкости.

## Что считается выполненным (на лист)

См. Definition of Done в [TEMPLATE_AUTHORING_GUIDE.md](TEMPLATE_AUTHORING_GUIDE.md):
заглушки удалены, авторские шаблоны дают тот же тип задач, ответы сверены, страж-тест
зелёный, ворклист помечен `done`, `WORK_LOG.md` обновлён.

## Порядок приоритетов

Начинать с тем, которые уже полностью поддержаны движком и легко проверяемы:
прямые вычисления (A), цифры/подсчёт (C), теория чисел и делимость (D). Геометрия
(G, H) и логика (K) — позже: там чаще нужны новые функции движка или solver'ы;
такие случаи агент помечает в `notes` и заводит функцию в `_FUNCTIONS`.

## Готовый промпт для запуска агента

Скопируй в новый чат/агент, подставив букву группы (пример — `D`):

```text
Ты работаешь в репозитории 15min (генератор задач-пятиминуток).

Задача: заменить bridge-заглушки авторскими шаблонами для ГРУППЫ D (Теория чисел).

Сначала прочитай:
- Docs/TEMPLATE_AUTHORING_GUIDE.md — как писать шаблоны (следуй ему буквально);
- Docs/AGENT_TASKS.md — правила координации;
- Docs/PROBLEM_TEMPLATES.md — обзор системы.

Затем:
1. Создай ветку authoring/D.
2. Возьми листья своей группы (leaf_id начинается с 'D') из
   data/source_index/per_task_template_worklist.json, у которых status = "todo".
   Открой соответствующие файлы data/source_index/task_tree/D_*/D**.md и оригиналы
   в Docs/all_tasks_all_files.md, чтобы понять тип каждой задачи.
3. Иди батчами по одному листу. Для каждого:
   - при необходимости добавь number_strategy (@_number_strategy) и функцию в
     _FUNCTIONS в problemgen/generation/template_generator.py (имена с префиксом темы);
   - напиши авторский шаблон(ы) и добавь в data/templates/problem_templates.json
     скриптом, одновременно удалив заглушки этого листа (source_tree_leaf == "D**");
   - сгенерируй несколько задач и СВЕРЬ ответ независимым расчётом;
   - пометь задачи листа в ворклисте: template_text/number_strategy/answer_formula/
     answer_type/notes и status = "done";
   - прогони python -m unittest discover -s tests (должно быть зелёно, включая
     test_no_template_is_fragile_across_seeds);
   - обнови Docs/WORK_LOG.md и сделай коммит на ветку authoring/D.
4. Не трогай листья и заглушки других групп. НЕ пушь в main напрямую — открывай PR
   из authoring/D.

Критерий готовности листа — Definition of Done из TEMPLATE_AUTHORING_GUIDE.md.
Работай маленькими коммитами. После каждого батча кратко отчитайся, сколько задач
переведено в done и что осталось.
```

## Текущий прогресс

- Ворклист: **34 / 2121 done** (батчи «частота цифры» C04 и «сравнение произведений» A04).
- Разблокирован движок: операторы `// % **`, функции (`gcd, count_digit, weekday_after,
  bigger_label`, …), типы ответа `multi`/`text`.
- Надёжность: 0 хрупких шаблонов, есть страж-тест.
