# База знаний агентов (TriliumNext Trilium)

Иерархическая накопительная память между агентами и разработчиками проекта.
Движок — [TriliumNext Trilium](https://github.com/TriliumNext/trilium):
дерево заметок, полнотекстовый поиск, веб-интерфейс для людей, REST ETAPI для
агентов. Инстанс — **локальный у каждого разработчика** (никаких сторонних
серверов); общая истина — markdown-зеркало `knowledge/` в git этого репозитория.

## Зачем БЗ при живом git

Git и `Docs/` — источник истины. БЗ не заменяет их, а экономит токены агентов:
вместо повторного изучения репозитория агент читает короткие карты-маршруты,
разборы регрессий и журнал. **При расхождении БЗ и репозитория верен
репозиторий**; заметки — краткий markdown со ссылками на файлы, без копий кода.

БЗ НЕ заменяет обязательные по `AGENTS.md` документы: `Docs/WORK_LOG.md`,
`Docs/FILE_INDEX.md`, `Docs/VECTOR_TREE.md` ведутся как раньше; в журнал БЗ
пишется одна краткая строка со ссылкой на подробную запись WORK_LOG.

## Запуск (локально, без docker)

```bash
node scripts/kb_server.mjs        # веб-UI: http://127.0.0.1:8481
```

Лаунчер сам находит собранный checkout Trilium (`TRILIUM_CHECKOUT`,
`../tools/trilium` или `../igrigin/tools/trilium`), при первом запуске создаёт
каталог данных `trilium-data-15min` рядом с checkout (config.ini: loopback,
`noAuthentication=true` — допустимо только для 127.0.0.1) и инициализирует БД.

Разовая подготовка checkout (вне репозитория проекта, Node >= 20 + pnpm):

```bash
git clone https://github.com/TriliumNext/trilium.git ../tools/trilium
cd ../tools/trilium && pnpm install --frozen-lockfile && pnpm run --filter server build
```

## CLI для агентов

```bash
node scripts/kb.mjs info                       # версия сервера
node scripts/kb.mjs tree "15min"               # разделы
node scripts/kb.mjs get "15min / 00 Обзор"
node scripts/kb.mjs put "15min / 30 Регрессии / <имя>" --stdin
node scripts/kb.mjs append "15min / 90 Журнал" "ветка X влита, тесты OK"
node scripts/kb.mjs search "хрупк"
```

Путь — сегменты через « / », сопоставление по точному названию заметки.

## Обязательная иерархия

- `15min / 00 Обзор` — что за проект, минимальный стартовый набор чтения;
- `15min / 10 Карта проекта` — слои, три маршрута авторинга, инварианты;
- `15min / 20 Семейства шаблонов` — карточка на семейство: файлы, ключевые
  функции, команда проверки;
- `15min / 30 Регрессии` — разбор каждого настоящего дефекта:
  «симптом → причина → фикс → чем защищено» со ссылками на файлы/коммиты;
- `15min / 40 Эксплуатация` — запуск сайта, печать, окружение, сама БЗ;
- `15min / 90 Журнал` — **append-only**: `- YYYY-MM-DD: <ветка>: <событие>`.

## Совместная работа через git (зеркало `knowledge/`)

Зеркало: ветка дерева → каталог (свой текст ветки → `_note.md`),
лист → `<название>.md`. Trilium у каждого свой; синхронизация — только через
git. `sync-in` создаёт и обновляет, но **ничего не удаляет** — удаление заметки
делается правкой `knowledge/` (+ вручную в своём Trilium при желании).

```bash
# начало работы (после git pull) — влить чужие изменения в свой Trilium:
node scripts/kb.mjs sync-in "15min" knowledge

# конец работы — выгрузить свои правки и закоммитить вместе с веткой:
node scripts/kb.mjs sync-out "15min" knowledge
git add knowledge && git commit
```

Параллельная работа двух разработчиков:

- правки в разных заметках сливаются git-ом без конфликтов;
- `90 Журнал` — только дописывание строк снизу; при конфликте слияния
  сохранить ОБЕ строки (порядок дат не важен);
- одну и ту же заметку в двух ветках одновременно не переписывать — раздел
  занят тем, кто указан в `Docs/AGENT_STATUS.md`.

## Правила для агентов

1. Перед работой: `git pull` → `sync-in` → прочитать `00 Обзор` и хвост журнала.
2. После завершённой задачи: обновить заметку затронутого раздела, дописать
   строку в `90 Журнал`, сделать `sync-out` и закоммитить `knowledge/` в ту же
   ветку, что и код.
3. Каждый настоящий дефект — отдельная заметка в `30 Регрессии`.
4. Запрещено класть в БЗ: секреты и токены, персональные данные, копии
   read-only корпуса `Docs/all_tasks_all_files.md`, длинные листинги кода.

## Проверка первого запуска на новой машине

1. `node scripts/kb_server.mjs` — сервер отвечает на
   `http://127.0.0.1:8481/api/health-check`;
2. `node scripts/kb.mjs info` — печатает версию;
3. `node scripts/kb.mjs sync-in "15min" knowledge` — дерево появилось в веб-UI;
4. `node scripts/kb.mjs append "15min / 90 Журнал" "машина <имя> подключена"`;
5. `node scripts/kb.mjs sync-out "15min" knowledge` и `git diff` — только
   ожидаемая строка журнала.
