# Система русскоязычных шаблонов задач

## Зачем это нужно

Наивная подстановка чисел в f-строки даёт неграмотный текст:

```
❌ "У {heads} голов и {legs} ног"  →  "У 100 голов и 101 ног"
✅ После движка:                   →  "У 100 голов и 101 ноги"
```

Проблема в том, что:
- числительное управляет падежом существительного: **1 нога / 2 ноги / 5 ног**
- существительные изменяются по падежам: **пасутся овцы** (им.), **у овец** (род.)
- при смене животного всё должно меняться автоматически

Решение — шаблонный движок с морфологическими слотами.

## Синтаксис слотов

Слот — это фрагмент `{...}` внутри строки шаблона.

| Слот | Что делает | Пример |
|---|---|---|
| `{key}` | Прямая подстановка `str(context[key])` | `{n}` → `"100"` |
| `{key:case}` | Падежная форма `RussianNoun` | `{a:gen_pl}` → `"овец"` |
| `{key:count,numkey}` | Число + согласованная форма | `{НОГ:count,n}` → `"100 ног"` |
| `{key:agree,numkey}` | Только согласованная форма | `{НОГ:agree,n}` → `"ног"` |

### Допустимые значения `case`

Единственное число:

| Код | Падеж | Пример |
|---|---|---|
| `nom` | Именительный | кот |
| `gen` | Родительный | кота |
| `dat` | Дательный | коту |
| `acc` | Винительный | кота |
| `ins` | Творительный | котом |
| `pre` | Предложный | коте |

Множественное число:

| Код | Падеж | Пример |
|---|---|---|
| `nom_pl` | Именительный | коты |
| `gen_pl` | Родительный | котов |
| `dat_pl` | Дательный | котам |
| `acc_pl` | Винительный | котов |
| `ins_pl` | Творительный | котами |
| `pre_pl` | Предложный | котах |

### Пример

```python
from problemgen.russian.template_engine import render_template
from problemgen.russian.noun_dict import NOUNS

text = render_template(
    "На {place} пасутся {a1:nom_pl} и {a2:nom_pl}. "
    "У {a1:gen_pl} и {a2:gen_pl} вместе {HEAD:count,h} и {LEG:count,l}.",
    {
        "place":  "лугах Простоквашино",
        "a1":     NOUNS["овца"],
        "a2":     NOUNS["курица"],
        "HEAD":   NOUNS["голова"],
        "LEG":    NOUNS["нога"],
        "h": 36,
        "l": 100,
    }
)
# → "На лугах Простоквашино пасутся овцы и курицы.
#    У овец и кур вместе 36 голов и 100 ног."
```

Проверка числительного + падежа:

| n | `{HEAD:count,n}` | `{ГОД:count,n}` | `{РАЗ:count,n}` |
|---|---|---|---|
| 1 | 1 голова | 1 год | 1 раз |
| 2 | 2 головы | 2 года | 2 раза |
| 5 | 5 голов | 5 лет | 5 раз |
| 11 | 11 голов | 11 лет | 11 раз |
| 21 | 21 голова | 21 год | 21 раз |
| 100 | 100 голов | 100 лет | 100 раз |
| 101 | 101 голова | 101 год | 101 раз |


## Структура RussianNoun

```python
@dataclass(frozen=True)
class RussianNoun:
    # Единственное число
    nom: str   # именительный — нога
    gen: str   # родительный  — ноги
    dat: str   # дательный    — ноге
    acc: str   # винительный  — ногу
    ins: str   # творительный — ногой
    pre: str   # предложный   — ноге
    # Множественное число
    nom_pl: str  # ноги
    gen_pl: str  # ног
    dat_pl: str  # ногам
    acc_pl: str  # ног
    ins_pl: str  # ногами
    pre_pl: str  # ногах
    # Свойства
    gender: str   # "m" / "f" / "n"
    animate: bool = False
```

Метод `count_forms()` возвращает `(nom, gen, gen_pl)` — тройку форм для `pluralize_ru()`:

```python
NOUNS["год"].count_forms()  # → ("год", "года", "лет")
# использование:
# pluralize_ru(74, ("год", "года", "лет"))  → "года"
# pluralize_ru(5,  ("год", "года", "лет"))  → "лет"
```

**Важно:** формы нестандартных парадигм заданы явно, не генерируются автоматически:
- `год` → мн. ч. род. п. `"лет"` (не `"годов"`)
- `курица` → мн. ч. род. п. `"кур"` (не `"куриц"`)
- `человек` → мн. ч. `"человек"` / `"людям"` (супплетивная парадигма)

## Шаблоны задач

### Шаблон 1 — heads_and_legs (Головы и ноги)

**Источник задач:** задачи 5, 188, 193 из списка

**Математика:**
```
x + y = total_heads
legs1·x + legs2·y = total_legs
──────────────────────────────
x = (total_legs - legs2·total_heads) / (legs1 - legs2)
y = total_heads - x
```

**Строка шаблона:**
```
"На {place} пасутся {a1:nom_pl} и {a2:nom_pl}. "
"У {a1:gen_pl} и {a2:gen_pl} вместе {HEAD:count,heads} "
"и {LEG:count,legs}. "
"Сколько {a1:gen_pl} и сколько {a2:gen_pl}?"
```

**Контекст:**
| Ключ | Тип | Описание |
|---|---|---|
| `place` | `str` | Локация в предл. падеже (напр. "лугах Простоквашино") |
| `a1`, `a2` | `RussianNoun` | Виды животных |
| `HEAD`, `LEG` | `RussianNoun` | Всегда `NOUNS["голова"]`, `NOUNS["нога"]` |
| `heads`, `legs` | `int` | Суммарное число голов и ног |

**Пример вывода:**
```
На лугах вблизи Простоквашино пасутся лошади и курицы.
У лошадей и кур вместе 14 голов и 42 ноги.
Сколько лошадей и сколько кур?
```
*Ответ: Лошадь: 7, курица: 7*


### Шаблон 2 — ratio_sum (Пропорция и сумма)

**Источник задач:** задача 14

**Математика:**
```
x = ratio · y
x + y = total
──────────────
y = total / (ratio + 1)
x = ratio · y
```

**Строка шаблона:**
```
"{name1} {v_sg} в {РАЗ:count,ratio} больше {THING:gen_pl}, чем {name2}, "
"а вместе они {v_pl} {THING:count,total}. "
"Сколько {THING:gen_pl} {v_sg} {name1} и сколько {name2}?"
```

**Контекст:**
| Ключ | Тип | Описание |
|---|---|---|
| `name1`, `name2` | `str` | Имена персонажей |
| `v_sg` | `str` | Глагол в прош. вр. ед.ч. (согл. с полом name1) |
| `v_pl` | `str` | Глагол в прош. вр. мн.ч. |
| `РАЗ` | `RussianNoun` | `NOUNS["раз"]` |
| `THING` | `RussianNoun` | Объект задачи |
| `ratio` | `int` | Кратность (≥2) |
| `total` | `int` | Суммарное количество |

**Пример вывода:**
```
Вася посадил в 4 раза больше яблок, чем Коля,
а вместе они посадили 200 яблок.
Сколько яблок посадил Вася и сколько Коля?
```
*Ответ: Вася: 160, Коля: 40*


### Шаблон 3 — age_friends (Возраст друзей)

**Источник задач:** задачи 97, 102

**Математика:**
```
sum_then = sum_now + n · delta
n = (sum_then - sum_now) / delta
```

**Строка шаблона:**
```
"Возраст нескольких {PERSON:gen_pl} в сумме составляет {ГОД:count,sum_now}. "
"Через {ГОД:count,delta} их общий возраст составит {ГОД:count,sum_then}. "
"Сколько {PERSON:gen_pl}?"
```

**Контекст:**
| Ключ | Тип | Описание |
|---|---|---|
| `PERSON` | `RussianNoun` | Тип людей (`друг`, `подруга`, `одноклассник`) |
| `ГОД` | `RussianNoun` | `NOUNS["год"]` |
| `sum_now` | `int` | Сумма возрастов сейчас |
| `delta` | `int` | Число лет вперёд |
| `sum_then` | `int` | Сумма возрастов через delta лет |

**Пример вывода:**
```
Возраст нескольких друзей в сумме составляет 36 лет.
Через 5 лет их общий возраст составит 66 лет.
Сколько друзей?
```
*Ответ: 6*


### Шаблон 4 — round_robin (Круговой турнир)

**Источник задач:** задачи 189, 194

**Математика:**
```
games = n · (n - 1) / 2   (сочетания C(n, 2))
```

**Строка шаблона:**
```
"В {tournament} участвуют {PERSON:count,n}. "
"Сколько {GAME:gen_pl} будет сыграно, "
"если каждый участник сыграет со всеми остальными ровно по одному разу?"
```

**Контекст:**
| Ключ | Тип | Описание |
|---|---|---|
| `tournament` | `str` | Название в предл. падеже ("шахматном турнире") |
| `PERSON` | `RussianNoun` | `NOUNS["человек"]` или `NOUNS["участник"]` |
| `GAME` | `RussianNoun` | `NOUNS["партия"]`, `NOUNS["матч"]` или `NOUNS["встреча"]` |
| `n` | `int` | Число участников |

**Пример вывода:**
```
В турнире по шашкам участвуют 4 человека.
Сколько партий будет сыграно, если каждый участник
сыграет со всеми остальными ровно по одному разу?
```
*Ответ: 6*


### Шаблон 5 — paint_cube (Окраска куба)

**Источник задач:** задачи 35, 130, 140, 145

**Математика:**
```
Площадь поверхности куба ∝ h²
answer = g_per_unit · h²
```

**Строка шаблона:**
```
"Чтобы покрасить поверхность деревянного {CUBE:gen} высотой 1 см "
"нужно {ГР:count,g} краски. "
"Сколько краски понадобится, чтобы покрасить деревянный {CUBE:acc} "
"высотой {h} см?"
```

**Контекст:**
| Ключ | Тип | Описание |
|---|---|---|
| `CUBE` | `RussianNoun` | `NOUNS["кубик"]` |
| `ГР` | `RussianNoun` | `NOUNS["грамм"]` |
| `g` | `int` | Расход краски на куб 1×1 см (граммов) |
| `h` | `int` | Высота целевого куба (см) |

**Пример вывода:**
```
Чтобы покрасить поверхность деревянного кубика высотой 1 см
нужно 14 граммов краски.
Сколько краски понадобится, чтобы покрасить деревянный кубик
высотой 3 см?
```
*Ответ: 126 граммов*  (14 × 3² = 126)


### Шаблон 6 — equal_payment (Совместная оплата)

**Источник задач:** задачи 107, 112

**Математика:**
```
Каждый должен заплатить total / 2.
Второй заплатил paid2, получив loan от первого.
debt = loan + total/2 - paid2
```

**Строка шаблона:**
```
"{name1} и {name2} хотят вместе заплатить {РУБ:count,total}, "
"разделив затраты поровну. "
"{name1} {v1_lend} {pron2_dat} взаймы {РУБ:count,loan}. "
"{name2} {v2_pay} {РУБ:count,paid2}, а {name1} — оставшиеся {РУБ:count,paid1}. "
"Сколько {РУБ:gen_pl} {v2_modal} вернуть {name2}?"
```

**Контекст:**
| Ключ | Тип | Описание |
|---|---|---|
| `name1`, `name2` | `str` | Имена персонажей |
| `v1_lend` | `str` | "дал"/"дала" (согл. с полом name1) |
| `v2_pay` | `str` | "заплатил"/"заплатила" (согл. с полом name2) |
| `v2_modal` | `str` | "должен"/"должна" (согл. с полом name2) |
| `pron2_dat` | `str` | "ему"/"ей" (согл. с полом name2) |
| `РУБ` | `RussianNoun` | `NOUNS["рубль"]` |
| `total`, `loan`, `paid2`, `paid1` | `int` | Суммы в рублях |

**Пример вывода:**
```
Надя и Алёша хотят вместе заплатить 2128 рублей, разделив затраты поровну.
Надя дала ему взаймы 9928 рублей.
Алёша заплатил 1284 рубля, а Надя — оставшиеся 844 рубля.
Сколько рублей должен вернуть Алёша?
```
*Ответ: 9708 рублей*


## JSON-формат вывода

Каждая задача — объект `ProblemRecord`, сериализуемый в JSON:

```json

  "code": "ARI-HLG-00001",
  "category": "arithmetic",
  "domain": "arithmetic",
  "template_name": "heads_and_legs",
  "problem_text": "На лугах вблизи Простоквашино пасутся лошади и курицы. У лошадей и кур вместе 14 голов и 42 ноги. Сколько лошадей и сколько кур?",
  "answer_text": "Лошадь: 7, курица: 7",
  "answer_values": [7, 7],
  "difficulty": {
    "level": "easy",
    "total_heads": 14,
    "total_legs": 42
  },
  "metadata": {
    "topic": "арифметика",
```

## Добавление нового шаблона

1. **Добавить существительные** в `noun_dict.py` если нужны новые слова.

2. **Написать строку шаблона** с морфологическими слотами:
   ```python
   _MY_TMPL = (
       "В {city} живут {PERSON:count,n} {PERSON:gen_pl}. "
       "Каждый {v_sg} по {THING:count,k} {THING:gen_pl} в {ГОД:count,years}."
   )
   ```

3. **Написать решатель** в `solvers.py` (чистая математика, без текста).

4. **Написать генераторную функцию** в `templates.py`:
   ```python
   def generate_my_template(*, rng, index, difficulty_level) -> ProblemRecord:
       # 1. Выбрать параметры (через rng)
       # 2. Вызвать решатель
       # 3. Сформировать контекст
       # 4. Вызвать render_template(...)
       # 5. Вернуть ProblemRecord
   ```

5. **Зарегистрировать** в `TEMPLATE_FACTORIES` и `list_templates()`.

### Правила написания шаблонов

- Глаголы с полом персонажа → хранить в контексте как строку: `"дал"/"дала"`
- Имена собственные не склоняются движком — используйте местоимения или избегайте косвенных падежей имён
- Нестандартные числительные (оба/обе, пол-/полу-) — вписывать напрямую
- Проверяйте граничные случаи: n=1, n=11, n=21, n=100
