# Доска занятости агентов (claim-board)

Этот файл — единственный источник правды о том, **какая группа сейчас в работе**,
чтобы два агента не взяли одну и ту же. Живёт в `main`.

## Протокол

1. **Занять (claim) — ДО начала работы.** Выбери группу со статусом `свободна`.
   Поменяй её статус на `в работе`, впиши ветку, дату и свой идентификатор,
   и сразу закоммить **только это изменение** в `main` (маленький claim-commit —
   единственное разрешённое прямое изменение `main`). После этого веди работу на
   своей ветке `authoring/<группа>`.
2. **Не трогать чужое.** Группу со статусом `в работе` не берёт никто другой.
3. **Освободить (release) — ПОСЛЕ вливания PR.** Когда все листья группы `done` и
   PR влит в `main`, поменяй статус на `готово (только перепроверка)`. Такую группу
   больше не переписывают — только перепроверяют.
4. **Прервался, не доделав?** Верни статус в `свободна` (или `частично — свободна`),
   чтобы группу мог подхватить другой агент. В `Заметки` напиши, что осталось.
5. **Есть ещё ресурсы?** Возьми следующую `свободную` группу и повтори. Цель —
   довести **все** группы до `готово`, а не только одну.

Статусы: `свободна` · `в работе` · `готово (только перепроверка)` ·
`частично — свободна`.
