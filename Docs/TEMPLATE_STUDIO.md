# Template Studio

`Template Studio` — локальная административная зона сайта для подготовки нового
параметрического шаблона. Маршрут: `/admin/template-studio`.

Это не генератор готовых production-шаблонов из произвольного текста. Он создаёт
редактируемый **draft**, показывает, что удалось детерминированно распознать, и
не даёт включить запись в worksheet до успешной валидации и явного действия
`Add to module`.

## Доступ и безопасность

- маршрут и API доступны только клиенту `127.0.0.1` или `::1`;
- для изменяющих API-запросов нужен одноразовый на процесс CSRF-токен, который
  получает только отрендеренная локальная страница;
- API принимает только JSON размером не более 64 KiB;
- исходное условие — только текст: оно не исполняется и не вставляется в DOM
  через `innerHTML`;
- формулы проходят AST-вычислитель `safe_expressions.py`; `eval`, `exec`,
  импорты, атрибуты, файлы, сеть и произвольные вызовы не поддерживаются.

Поскольку в сайте нет существующей системы аутентификации, loopback-ограничение
и CSRF — безопасное локальное ограничение, а не имитация входа по паролю.

## Поток работы и статусы

```text
анализ → draft → validated → active → archived
                 ↘ rejected
draft + подтверждённое Delete → удалён
```

- `draft` — обычный редактируемый черновик; его можно окончательно удалить,
  только подтвердив действие;
- `validated` — все обязательные проверки прошли; запись ещё не участвует в
  генерации;
- `active` — явно опубликована в target module; hard-delete запрещён;
- `archived` — выведена из overlay-каталога, не генерируется и может быть
  восстановлена лишь через повторную валидацию;
- `rejected` — сохранённый черновик с причиной отклонения.

## Что анализатор умеет и чего не обещает

`TemplateAnalyzer` детерминированно выделяет числа, русские имена с заглавной
буквы, единицы, обычные арифметические знаки и простые выражения. Для очевидной
арифметической последовательности с многоточием он предлагает независимые
`first_term`, `difference`, `term_count`, derived-значения и формулу суммы.

Для остальных текстов исходная формулировка сохраняется, найденные числа могут
стать черновыми параметрами, а в draft добавляется предупреждение. Доказательства,
чертежи, сложная семантика и произвольные сюжетные задачи не считаются понятыми:
их template text, параметры и solver strategy заполняются вручную. Studio не
склоняет имена и не пытается угадывать род; для именных заданий нужны явные
данные общего языкового слоя.

## Редактор, параметры и формулы

Редактор содержит отдельные секции Source, Template text, Parameters, Derived
values, Solver, Constraints, Grammar, Preview, Validation и History. Обычные
параметры редактируются структурированными полями; Raw JSON предназначен только
для расширенного редактирования и всё равно проходит серверную проверку.

Поддерживаемые типы независимых параметров:

`integer`, `positive_integer`, `nonnegative_integer`, `decimal`, `fraction`,
`boolean`, `string`, `word`, `letter`, `name`, `choice`, `integer_list`,
`word_list`.

Derived-значения вычисляются после независимых параметров и никогда не
выбираются случайно. Поддерживаемые типы ответа: `integer`, `number`,
`decimal`, `fraction`, `boolean`, `text`, `list`.

Белый список формул: `+`, `-`, `*`, `/`, `//`, `%`, `**` (степень 0–64),
`factorial`, `gcd`, `lcm`, `abs`, `min`, `max`, `sum`, скобки, списки и только
явно объявленные переменные. Деление на ноль, неизвестные переменные, циклы
derived-значений и слишком широкие диапазоны отвергаются.

## Проверка и публикация

Валидация показывает отдельные результаты для структуры JSON, уникальности ID,
target module, стратегии, плейсхолдеров, диапазонов, формул, ученического текста,
базовой русской пунктуации и десяти фиксированных seed. Для каждого seed
проверяются bounded generation, вычисление derived-значений, вычислимый ответ,
соответствие answer type и повторный расчёт ответа. Неразрешённые плейсхолдеры,
division by zero и нецелый ответ при `integer` блокируют публикацию.

`Add to module` создаёт active overlay-запись через структурированный JSON,
проверяет ID ещё раз и сохраняет атомарный backup до `os.replace`. Сайт добавляет
активный шаблон к внутреннему выбору его target module; он появляется и в
метаданных селектора. Базовые каталоговые JSON модулей не переписываются
строковой заменой и незавершённые drafts в них не попадают.

## Хранение

Все runtime-данные находятся в `data/template_studio/`:

- `drafts/<draft_id>.json` — полный draft с исходным текстом, анализом,
  параметрами, формулами, source metadata, revision history и последним report;
- `validation_reports/<draft_id>.json` — подробный последний результат;
- `history/<draft_id>.json` — аудит действий (`draft_created`, `analysis_run`,
  `draft_edited`, `preview_generated`, `validation_*`, `activated`, `archived`,
  `restored`, `rejected`, `deleted`);
- `history/catalogue_before_*.json` — снимки active overlay до изменения;
- `active_templates.json` — только active записи, которые читает сайт.

## Ограничения первой версии

Studio не является CAS, не решает произвольные олимпиадные задачи и пока не
выполняет автоматическую эволюцию действующих шаблонов. Независимый расчёт
проверяет безопасную формулу на наборах generated values; для новой предметной
стратегии нужно добавить отдельную детерминированную реализацию и тесты, а не
маскировать её текстовой формулой.
